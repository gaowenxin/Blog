J2SE之JAVA概述

java两种核心机制： java虚拟机,垃圾收集机制

java虚拟机： 源代码文件(＊.java)经过java编译器翻译成字节码(＊.class)，通过类装载器，字节码校验器，解释器交给操作系统平台，操作系统把它翻译成机器可以认知的部分。

操作系统上方的环境是给java编译执行的，叫做java虚拟机，以java为代码作为可执行文件。不同的平台有不同的虚拟机，正因为如此，java才真正实现了跨平台，在任何一个地方的写的java文件，首先把它编译成class文件，当它要执行的时候，如果放在UNIX上，它就在UNIX上面的虚拟机上执行，如果放在windows上，它就在windows上面的虚拟机上执行，这些虚拟的环境对于class文件的接口是一样的，可以实现一次编译到处运行。

垃圾收集机制：不需要程序员负责内存空间的回收

JDK:Software Development Kit(软件开发包)

JDK:Java Runtime Environment(java运行环境)

开发需要JDK,用户只需JDK

Java Application初步

  • java源文件以“java”为扩展名。源文件的基本组成部分是类（class），如本例中的HelloWorld类
  • 一个源文件中最多只有一个public类。其他类的个数不限，如果源文件包含一个public类，它必须按该类命名
  • Java应用程序的执行入口是main()方法。固定书写格式：public static void main(String args[ ]){…}

标识符

凡是自己可以取名字的地方都叫标识符

命名规则：

  • 标识符由字母，数字，下划线”_”，美元符”$”组成
  • 不能以数字开头
  • 标识符大小写敏感，长度无限制

关键字

Java的关键字都是小写

Java常量

  • 整型常量：123
  • 实型常量：3.14
  • 字符常量：’a’
  • 逻辑常量：true,false
  • 字符串常量：”hello world”

Java变量

从本质上讲，变量其实是内存中的一小块区域，使用变量名来访问这块区域，每个变量名使用之前必要要申请（声明），然后进行填充内容，才能使用（赋值）

程序执行过程

.exe直接放到操作系统上执行.class放到虚拟机上执行

在硬盘上会有一个程序存在，比如.exe文件，比如.class文件，区别不是很大。但是这个时候只是一个静态的概念，想要运行起来，首先把程序代码load到内存的区域，操作系统会找到代码，开始执行，执行的过程中会把内存主要分为四块区域：code segment(存放代码)，data segment(静态变量 字符串常量)，stack(局部变量)，heap(new出来的东西)

Java变量的分类

  • 按被声明的位置划分：局部变量，成员变量

局部变量：方法或语句块内部定义的变量

成员变量：方法外部，类的内部定义的变量（类外面不能有变量的声明）

Java基本数据类型

  • 逻辑型：boolean
  • 字符型：char
  • 整数型：byte，short，int，long
  • 浮点数型：float，double

逻辑型Boolean:boolean类型只允许取值true或false，不可以用0或非0的整数代替true或false

字符型char:字符常量为用单引号括起来的单个字符

整数类型：Java的整数类型不受具体操作系统的影响，保证了 Java程序的可移植性

类型 占用存储空间 表数范围
byte 1字节 -128～127
short 2字节
int 4字节 -2的31次幂～2的31次幂减一
long 8字节

浮点数类型：Java的浮点数类型不受具体操作系统的影响，保证了 Java程序的可移植性

类型 占用存储空间 表数范围
float 4字节 -3.403E38～3.403E38
double 8字节 -1.798E308～1.798E308
